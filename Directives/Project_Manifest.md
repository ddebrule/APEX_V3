**Last Updated:** 2026-01-17

## System Goal
Increase the probability of sportsman and intermediate racers making the podium in **1/8 Scale Nitro and Eco Off-Road racing.**

## Racer Profile Architecture (Multi-Racer Capable)
- **Racer Identity:** Name, Email, Socials (Facebook/Instagram).
- **Fleet Management:** Multi-car registry (Brand, Model, Transponder ID).
- **Sponsor Management:** Dynamic list of sponsor companies.

## Mandatory Dashboard Architecture (The 6-Tab Flow)
1. **Tab 1: Mission Control (Pre-Event Strategy)**
   - "Where are we going and are we ready?"
2. **Tab 2: The Pit Lane (Race Day Context)**
   - "What are the track conditions RIGHT NOW?"
3. **Tab 3: The Driver Stand (Debrief Input)**
   - "What is the car doing wrong?"
4. **Tab 4: The Engineer (Diagnosis & Fix)**
   - "How do I fix the problem?"
5. **Tab 5: The Scoreboard (Validation)**
   - "Did the last change work?"
6. **Tab 6: The Garage (Configuration)**
   - "What is the exact reference setup?"

## Core Feature Pillars
- **Institutional Memory System:** Learning from historical track history and condition history.
- **Race Prep Plan Generator:** Proactive PDF preparation for upcoming events.
- **Hybrid Parsing Engine:** High-fidelity setup sheet extraction (PDF/Vision).
- **Optimized Scribe:** Keyword-based signal detection for high-stress input.

## Rebuild Blueprint: A.P.E.X. V3
- **Self-Annealing Loop:** Analyze, Fix, Learn, Update.
- **Living Directives:** Knowledge persistence in Layer 1.
- **Design OS Workflow:** Spec First -> Design First -> Code Last.
